The bouncer exports Prometheus metrics when `metrics.enabled` is `true`. Metrics are available at `http://<listen_addr>:<listen_port>/metrics`.

## Bouncer Metrics

### `crowdsec_routeros_active_decisions`

<Badge text="Gauge" variant="note" /> <Badge text="Labels: origin, action, scope, ip_type" variant="default" />

Number of currently active (banned) decisions, broken down by source, action, scope, and IP type.

| Label | Values | Description |
|-------|--------|-------------|
| `origin` | `crowdsec`, `cscli`, `CAPI`, `lists:*` | Source of the decision |
| `action` | `ban` | Action type (currently only `ban`) |
| `scope` | `Ip`, `Range` | Single IP or CIDR range |
| `ip_type` | `ipv4`, `ipv6` | Protocol version |

```txt
# Total active decisions
sum(crowdsec_routeros_active_decisions)

# Active decisions by origin
sum by (origin) (crowdsec_routeros_active_decisions)

# Only local decisions (non-CAPI)
sum(crowdsec_routeros_active_decisions{origin!="CAPI"})

# IPv6 decisions
sum(crowdsec_routeros_active_decisions{ip_type="ipv6"})
```

---

### `crowdsec_routeros_processed_total`

<Badge text="Counter" variant="success" /> <Badge text="Labels: action, origin, ip_type" variant="default" />

Total number of decisions processed since startup. Counts both bans (`action="ban"`) and unbans (`action="unban"`).

```txt
# Ban rate per minute by origin
rate(crowdsec_routeros_processed_total{action="ban"}[5m]) * 60

# Total unbans since startup
sum(crowdsec_routeros_processed_total{action="unban"})
```

---

### `crowdsec_routeros_lapi_calls_total`

<Badge text="Counter" variant="success" /> <Badge text="Labels: status" variant="default" />

Total LAPI API calls. Labels: `status="success"` or `status="error"`.

```txt
# LAPI error rate
rate(crowdsec_routeros_lapi_calls_total{status="error"}[5m])
```

---

### `crowdsec_routeros_routeros_calls_total`

<Badge text="Counter" variant="success" /> <Badge text="Labels: status" variant="default" />

Total RouterOS API calls. Labels: `status="success"` or `status="error"`.

```txt
# RouterOS error rate
rate(crowdsec_routeros_routeros_calls_total{status="error"}[5m])
```

---

### `crowdsec_routeros_stream_events_total`

<Badge text="Counter" variant="success" /> <Badge text="Labels: status" variant="default" />

Total stream events received from LAPI. Each event contains 0+ decisions. Labels: `status="success"` or `status="error"`.

---

### `crowdsec_routeros_lapi_stream_latency_seconds`

<Badge text="Gauge" variant="note" />

Last recorded LAPI polling latency in seconds. Useful for detecting LAPI connectivity issues.

---

### `crowdsec_routeros_reconciliation_total`

<Badge text="Counter" variant="success" /> <Badge text="Labels: status" variant="default" />

Total reconciliation runs. Labels: `status="success"` or `status="error"`.

---

### `crowdsec_routeros_reconciliation_duration_seconds`

<Badge text="Gauge" variant="note" />

Duration of the last reconciliation run in seconds.

## RouterOS System Metrics

When `metrics.routeros_poll_interval` is non-zero, the bouncer also collects MikroTik system metrics:

### `crowdsec_routeros_system_cpu_load`

<Badge text="Gauge" variant="note" /> <Badge text="Labels: core" variant="default" />

CPU load percentage per core.

### `crowdsec_routeros_system_memory_total_bytes`

<Badge text="Gauge" variant="note" />

Total system memory in bytes.

### `crowdsec_routeros_system_memory_used_bytes`

<Badge text="Gauge" variant="note" />

Used system memory in bytes.

### `crowdsec_routeros_system_temperature_celsius`

<Badge text="Gauge" variant="note" /> <Badge text="Labels: component" variant="default" />

System temperature in Celsius.

```txt
# Memory usage percentage
crowdsec_routeros_system_memory_used_bytes / crowdsec_routeros_system_memory_total_bytes * 100

# Average CPU load across all cores
avg(crowdsec_routeros_system_cpu_load)
```

## LAPI Metrics

The bouncer collects CrowdSec LAPI metrics via the `/metrics` endpoint. These are proxied as gauges:

| Metric | Type | Description |
|--------|------|-------------|
| `crowdsec_routeros_lapi_active_decisions` | <Badge text="Gauge" variant="note" /> | Total active decisions in LAPI |
| `crowdsec_routeros_lapi_bouncers` | <Badge text="Gauge" variant="note" /> | Number of registered bouncers |
| `crowdsec_routeros_lapi_machines` | <Badge text="Gauge" variant="note" /> | Number of registered machines |
| `crowdsec_routeros_lapi_alerts_total` | <Badge text="Gauge" variant="note" /> | Total alerts in LAPI |
| `crowdsec_routeros_lapi_decisions_added_total` | <Badge text="Gauge" variant="note" /> | Decisions added counter from LAPI |
| `crowdsec_routeros_lapi_decisions_deleted_total` | <Badge text="Gauge" variant="note" /> | Decisions deleted counter from LAPI |

:::note
LAPI metrics require the CrowdSec LAPI to have its own Prometheus endpoint enabled and accessible.
:::
