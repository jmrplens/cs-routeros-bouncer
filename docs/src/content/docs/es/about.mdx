cs-routeros-bouncer es un componente de remediación (bouncer) libre y de código abierto de [CrowdSec](https://www.crowdsec.net/) para [MikroTik RouterOS](https://mikrotik.com/). Sincroniza las decisiones de ban/unban de CrowdSec con las reglas de firewall de RouterOS a través de la API de RouterOS, con reconciliación, métricas de Prometheus y limpieza segura. El proyecto está escrito en Go y publicado bajo la [Licencia MIT](https://github.com/jmrplens/cs-routeros-bouncer/blob/main/LICENSE).

## Cómo se prueba

La suite de pruebas abarca las capas unitaria, de integración y funcional (consulta la [Guía de pruebas](/cs-routeros-bouncer/es/development/testing-guide/)). Las cifras de rendimiento publicadas en esta documentación se midieron en hardware real:

| Componente       | Probado con                                          |
| ---------------- | ---------------------------------------------------- |
| Router           | MikroTik RB5009UG+S+ (ARM64, 4 núcleos, 1 GB de RAM) |
| RouterOS         | 7.22.1 (requiere 7.x con el servicio API habilitado) |
| CrowdSec         | 1.5+ con la Local API (LAPI)                         |
| Host del bouncer | Linux (amd64/arm64), Go 1.26.4                       |

Consulta [Benchmarking](/cs-routeros-bouncer/es/development/benchmarking/) para la metodología y las cifras completas.
