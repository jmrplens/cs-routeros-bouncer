cs-routeros-bouncer is a free, open-source [CrowdSec](https://www.crowdsec.net/) remediation component (bouncer) for [MikroTik RouterOS](https://mikrotik.com/). It syncs CrowdSec ban/unban decisions into RouterOS firewall rules through the RouterOS API, with reconciliation, Prometheus metrics, and safe cleanup. The project is written in Go and released under the [MIT License](https://github.com/jmrplens/cs-routeros-bouncer/blob/main/LICENSE).

## How it is tested

The test suite spans unit, integration, and functional layers (see the [Testing Guide](/cs-routeros-bouncer/development/testing-guide/)). Performance figures published in this documentation were measured on real hardware:

| Component    | Tested with                                        |
| ------------ | -------------------------------------------------- |
| Router       | MikroTik RB5009UG+S+ (ARM64, 4 cores, 1 GB RAM)    |
| RouterOS     | 7.22.1 (requires 7.x with the API service enabled) |
| CrowdSec     | 1.5+ with the Local API (LAPI)                     |
| Bouncer host | Linux (amd64/arm64), Go 1.26.4                     |

See [Benchmarking](/cs-routeros-bouncer/development/benchmarking/) for the methodology and full numbers.
