Settings for the CrowdSec LAPI connection and decision filtering.

## Connection

### `crowdsec.api_url` <Badge text="Required" variant="danger" />

**Env:** `CROWDSEC_URL` ôñ **Default:** `http://localhost:8080/`

The URL of the CrowdSec Local API (LAPI). Include the trailing slash.

```yaml
crowdsec:
  api_url: "http://localhost:8080/"
```

### `crowdsec.api_key` <Badge text="Required" variant="danger" />

**Env:** `CROWDSEC_BOUNCER_API_KEY` ôñ **Default:** ã

The bouncer API key. Generate one with:

```bash
sudo cscli bouncers add cs-routeros-bouncer
```

### `crowdsec.retry_initial_connect` <Badge text="Optional" variant="note" />

**Env:** `CROWDSEC_RETRY_INITIAL_CONNECT` ôñ **Default:** `true`

When enabled, the bouncer retries connecting to LAPI on startup if the initial connection fails. Useful when the bouncer starts before CrowdSec is ready.

## Polling

### `crowdsec.update_frequency` <Badge text="Optional" variant="note" />

**Env:** `CROWDSEC_UPDATE_FREQUENCY` ôñ **Default:** `10s`

How often to poll LAPI for new or expired decisions. Uses Go duration format (e.g., `10s`, `1m`, `30s`).

:::tip
Lower values provide faster response but increase LAPI load. `10s` is a good balance for most setups.
:::

### `crowdsec.reconciliation_interval` <Badge text="Optional" variant="note" />

**Env:** `CROWDSEC_RECONCILIATION_INTERVAL` ôñ **Default:** `15m`

How often to fetch a full active-decision snapshot from LAPI and reconcile it with MikroTik address-list membership. This repairs drift when an address-list entry is removed manually or expires on the router while CrowdSec still considers the decision active.

`crowdsec.reconciliation_interval` / `CROWDSEC_RECONCILIATION_INTERVAL` uses Go duration format, such as `1m`, `15m`, or `1h`.

Set to `0` to disable periodic reconciliation. Any non-zero value must be at least `1m`.

```yaml
crowdsec:
  reconciliation_interval: "15m"
```

### `crowdsec.lapi_metrics_interval` <Badge text="Optional" variant="note" />

**Env:** `CROWDSEC_LAPI_METRICS_INTERVAL` ôñ **Default:** `15m`

How often to report usage metrics to the CrowdSec LAPI `/v1/usage-metrics` endpoint. Set to `0` to disable.

Each push includes:

- **Active decisions** ã per-origin (e.g., `crowdsec`, `cscli`, `CAPI`) and per-IP-type (`ipv4`, `ipv6`)
- **Dropped traffic** ã bytes and packets blocked by MikroTik firewall rules (delta since last push)
- **Bouncer metadata** ã type, version, OS info, uptime

This data appears in the CrowdSec Console and helps track bouncer effectiveness.

```yaml
crowdsec:
  lapi_metrics_interval: "15m"
```

## Decision Filtering

### `crowdsec.origins` <Badge text="Optional" variant="note" />

**Env:** `CROWDSEC_ORIGINS` ôñ **Default:** `[]` (all origins)

Filter decisions by their origin. Empty means all decisions are processed.

| Origin     | Description                                    |
| ---------- | ---------------------------------------------- |
| `crowdsec` | Decisions from CrowdSec detection engine       |
| `cscli`    | Manual decisions via `cscli decisions add`     |
| `CAPI`     | Community blocklists from CrowdSec Central API |

```yaml
# Only local decisions (no community blocklists)
crowdsec:
  origins: ["crowdsec", "cscli"]
```

:::note[Local-only mode]
Setting `origins: ["crowdsec", "cscli"]` is the recommended way to exclude community blocklists (CAPI). This avoids pushing 20,000+ IPs to the router.
:::

### `crowdsec.scopes` <Badge text="Optional" variant="note" />

**Env:** `CROWDSEC_SCOPES` ôñ **Default:** `["ip", "range"]`

Decision scopes to process. Supported values: `ip`, `range`.

### `crowdsec.supported_decisions_types` <Badge text="Optional" variant="note" />

**Env:** `CROWDSEC_DECISIONS_TYPES` ôñ **Default:** `["ban"]`

Only decisions of these types are processed.

:::caution
This bouncer only implements the **`ban`** action, which translates to drop/reject firewall
rules on MikroTik. Other CrowdSec decision types such as `captcha` or `throttle` are not
applicable to a network firewall bouncer and will be silently ignored even if listed here.
:::

### `crowdsec.scenarios_containing` <Badge text="Optional" variant="note" />

**Env:** `CROWDSEC_SCENARIOS_CONTAINING` ôñ **Default:** `[]` (no filter)

Only process decisions from scenarios whose name contains one of these strings. Empty means no filtering.

```yaml
crowdsec:
  scenarios_containing: ["ssh", "http"]
```

### `crowdsec.scenarios_not_containing` <Badge text="Optional" variant="note" />

**Env:** `CROWDSEC_SCENARIOS_NOT_CONTAINING` ôñ **Default:** `[]` (no filter)

Exclude decisions from scenarios whose name contains one of these strings.

## TLS Authentication

For mutual TLS authentication with the LAPI:

### `crowdsec.cert_path` <Badge text="Optional" variant="note" />

**Env:** `CROWDSEC_CERT_PATH` ôñ **Default:** ã

Path to the TLS client certificate (PEM format).

### `crowdsec.key_path` <Badge text="Optional" variant="note" />

**Env:** `CROWDSEC_KEY_PATH` ôñ **Default:** ã

Path to the TLS client key (PEM format).

### `crowdsec.ca_cert_path` <Badge text="Optional" variant="note" />

**Env:** `CROWDSEC_CA_CERT_PATH` ôñ **Default:** ã

Path to the CA certificate (PEM format) for verifying the LAPI server certificate.

### `crowdsec.insecure_skip_verify` <Badge text="Optional" variant="note" />

**Env:** `CROWDSEC_INSECURE_SKIP_VERIFY` ôñ **Default:** `false`

Skip TLS certificate verification for LAPI connections.

:::danger
Only use in development/testing. Never in production.
:::
